Теория

CSS начинается с селекторов. Они позволяют выбрать нужные элементы для стилизации. На этой неделе ты изучишь:

  • Базовые селекторы: тег, класс, id, универсальный
  • Комбинированные селекторы: ul li, div.main
  • Атрибутные селекторы: input[type="text"]
  • Псевдоклассы: :hover, :active, :nth-child(2n)
  • Псевдоэлементы: ::before, ::after
  • Цветовые форматы: HEX, RGB, HSL, имена цветов
  • Каскадность и специфичность: как браузер выбирает стиль
  • Сброс стилей (reset): normalize.css

  • Что такое наследование и почему не все свойства наследуются
  • Как работает порядок подключения стилей
  • Вложенность селекторов и её влияние на структуру CSS
  • Почему важно использовать сброс стилей

Практические моменты

  • Используй devtools для изучения применённых стилей
  • Попробуй комбинировать селекторы для точечного воздействия

Теория

Flexbox — мощный инструмент для создания адаптивных и гибких макетов. Позволяет легко выравнивать, распределять и перестраивать элементы в контейнере.

  • display: flex — превращает контейнер в flex-контейнер.
  • flex-direction — направление главной оси (row, column и др.).
  • justify-content — выравнивание по главной оси (center, space-between и др.).
  • align-items — выравнивание по поперечной оси (center, flex-end и др.).
  • gap — расстояние между элементами.
  • flex-wrap — перенос строк (wrap).
  • flex-grow, flex-shrink, flex-basis — управление размером элементов.
  • order — изменение порядка элементов.
  • align-self — индивидуальное выравнивание элемента.
  • Flexbox отлично подходит для меню, карточек, футеров, галерей.

Дополнительная теория

  • Flexbox работает в одном измерении (строка или колонка).
  • Можно вкладывать flex-контейнеры друг в друга для сложных макетов.
  • Используй min-width: 0 для предотвращения переполнения flex-элементов.
  • Flexbox — основа для современных CSS-фреймворков.

Лайфхаки

  • Используй gap вместо margin для отступов между flex-элементами.
  • Для сложных сеток изучи CSS Grid.
  • Проверяй выравнивание на разных разрешениях экрана.
  • Используй align-items: stretch для равной высоты блоков.

Пример только для этой недели:

  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 16px;
}
.flex-demo div {
  background: #39ff14;
  color: #232d25;
  padding: 12px 24px;
  border-radius: 10px;
  font-weight: bold;
}

Теория

Анимации и переходы делают интерфейс живым и отзывчивым. transition позволяет плавно изменять свойства, а @keyframes — создавать сложные анимации.

  • transition: плавное изменение свойств (background, color, transform и др.).
  • transition-property, transition-duration, transition-timing-function, transition-delay
  • transform: scale, rotate, translate, skew
  • @keyframes — создание собственных анимаций.
  • animation: animation-name, animation-duration, animation-iteration-count, animation-direction, animation-fill-mode
  • Комбинирование анимаций и переходов.
  • События :hover, :focus, :active для запуска анимаций.
  • Аппаратное ускорение: will-change
  • Плавность: cubic-bezier для сложных кривых анимации.
  • Анимация SVG и псевдоэлементов.
  • Ограничения: не все свойства можно анимировать (например, display).

Лайфхаки

  • Не переборщи с анимациями — они должны помогать, а не отвлекать.
  • Используй will-change для оптимизации производительности.
  • Для сложных анимаций попробуй Animate.css или GSAP.
  • Проверяй, как анимации работают на мобильных устройствах.

Пример только для этой недели:

button {
  transition: background 0.3s, transform 0.2s;
}
button:hover {
  background: #39ff14;
  color: #232d25;
  transform: scale(1.05);
}
@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.block-animate {
  animation: fadeIn 1s ease;
}

Теория

Адаптивный дизайн — это подход, при котором сайт хорошо выглядит и работает на любых устройствах. Для этого используются медиа-запросы и гибкие единицы измерения.

  • @media — основной инструмент для адаптива.
  • max-width и min-width — условия для разных экранов.
  • em, rem, %, vw, vh — гибкие единицы.
  • Гибкие изображения: max-width: 100%;
  • Мобильное меню (бургер-меню).
  • Проверка адаптива через DevTools.
  • Контейнерные запросы (container queries) — новая возможность для truly responsive компонентов.
  • Mobile-first vs Desktop-first подходы.
  • Использование clamp() для адаптивных размеров.
  • Скрытие/отображение элементов на разных экранах.

Лайфхаки

  • Планируй адаптивность заранее.
  • Проверяй сайт на разных устройствах и разрешениях.
  • Используй относительные единицы для гибкости.
  • Не забывай про доступность: крупные кнопки, читаемый текст.
  • Проверь, чтобы интерактивные элементы были удобны для пальцев.

Пример только для этой недели:

@media (max-width: 600px) {
  body {
    font-size: 0.9rem;
    padding: 8px;
  }
  .main-btn {
    width: 100%;
    font-size: 1rem;
  }
}

  • Изменение темы через переменные (например, светлая/тёмная тема)
  • Переменные для размеров, отступов, шрифтов
  • Лучшие практики: структура CSS, комментарии, переиспользование кода
  • Модули CSS (BEM, SMACSS, ITCSS) — подходы к организации классов
  • Инструменты: препроцессоры (Sass, Less), автопрефиксер
  • Каскадность переменных: можно переопределять переменные в разных селекторах
  • Использование переменных в медиа-запросах и анимациях
  • Сравнение CSS-переменных и переменных препроцессоров
  • Валидация и поддержка переменных в разных браузерах

Финальный проект: твой современный сайт

Задача: Создай или доработай свой сайт, используя все техники из курса.
Требования:

  • Используй CSS-переменные для цветов, размеров, отступов.
  • Сделай сайт адаптивным с помощью медиа-запросов и гибких единиц (rem, vw, %).
  • Добавь анимации и плавные переходы для кнопок, ссылок, блоков.
  • Используй Flexbox для построения макета и выравнивания.
  • Оформи заголовки, текст, меню и подвал согласно лучшим практикам.
  • Добавь переключение темы (светлая/тёмная) через CSS-переменные.
  • Прокомментируй свой CSS и используй BEM-нейминг для классов.
